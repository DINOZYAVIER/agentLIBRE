agentLIBRE name, logos, domains, package names, release identities, hosted
services, or other branding in a way that suggests official status,
sponsorship, certification, or endorsement.

You may truthfully state that your work is based on agentLIBRE, compatible with
agentLIBRE, or a fork of agentLIBRE, as long as the statement is not misleading.

Do not describe modified versions, forks, packages, services, or builds as
official agentLIBRE releases unless you have separate permission.

Official infrastructure, if any, is separate from the code license. This may
include domains, hosted services, update channels, release signing identities,
package namespaces, app store listings, or similar project infrastructure.

This policy may be expanded later as the project develops.
